快速获取图片宽高
  • 2014-09-25
  • JavaScript

开发过一个等宽瀑布流，请求后端的数据发现后端并没有提供图片宽高，应该是后端的哥们忙吧。加之项目比较赶，还是想办法在前端解决吧。

那时的解决方法就是直接监听请求来的每一组的数据的图片的onload事件，触发一次或者img.complete === true就记一次数，发现加载完了，就把图片按照瀑布流的布局显示出来。因为无法预知每张图片到底高度多少，所以不知道瀑布流接下来该怎么摆，所以整个过程是阻塞的，也就是图片不加载完，就不干其他事。

Zepto中$.fn.each的一个要注意的地方
  • 2014-09-23
  • JavaScript
  • Zepto 

2014-09-25补充：
Zepto的文档中其实是对return false有说明的：

Iterate through every element of the collection. Inside the iterator function, this keyword refers to the current item (also passed as the second argument to the function). If the iterator function returns false, iteration stops.

是我疏忽了。不过可以把本文当做$.fn.each方法实现的探讨吧。

原文：

前几天在用Zepto做一个项目，同事写了一个小插件，代码的作用是找到某个父元素下的所有img标签，然后将其中lazyload属性的值赋给src属性，达到控制加载图片的目的。

首先想到的，当然是这样实现：

    
        $('img').each(function(idx, el){
            if(!$(el).attr('lazyload')) return false;
            // do something here
        });

看代码第二行的return false，这行会导致执行流跳出each循环。

transform的3d变幻
  • 2014-09-17
  • CSS

当自己觉得有些东西在脑袋里躺得好好的呢，有啥忘的？这时，其实已经晚了。—— 佚名

我们在看这个世界，看2D是在看，看它在离我们50米开外一片幕布上变幻。甚至我们可以走到它跟前不到一米处看，因为你知道幕布永远和视线垂直，不会有东西能从那块幕布里蹦出来袭向我们的。其实，在2D的世界里，并没有一个「我们」的视角站在一旁观看，2D就是2D，幕布里的变幻，就是这个世界的全部了。

但是3D就不一样了。回想下电影贞子里鬼从电视里钻出来的画面。我们觉得很恐怖，是因为2D世界被打破了，开始有东西袭向我们了。又比如，2012年重新制作的3D版泰坦尼克号，为什么前排的同学情不自禁的向前摸去？因为……不说了，连卡梅隆都忍不住说了，那个3D的效果太逼真了。所以，3D就是多了一根坐标轴，方向和前排同学伸出去的手平行。
